Open-Source Email Infrastructure Monitoring

Detect hung mail servers, validate SMTP/IMAP connectivity, and ensure reliable mail delivery. Self-hosted, production-ready, Docker native.

View on GitHub Quick Start

How It Works

1

Configure

Set your mail server credentials and external test accounts via environment variables or Docker secrets.

2

Deploy

Run with Docker, Docker Compose, Kubernetes, or plain Node.js. Works anywhere containers run.

3

Monitor

Point UptimeRobot, Prometheus, or any HTTP monitor at the /status endpoint. Get instant alerts on failures.

# Pull from GitHub Container Registry
docker pull ghcr.io/0xa1bed0/mail-healthcheck:latest

# Run with your configuration
docker run -d -p 3000:3000 \
  -e LOCAL_SMTP_HOST=smtp.your-server.com \
  -e LOCAL_SMTP_USER=monitor@your-server.com \
  -e EXT_SMTP_HOST=smtp.gmail.com \
  ghcr.io/0xa1bed0/mail-healthcheck:latest

# Check status
curl http://localhost:3000/status

Features

üîç

Detects Hung Servers

Catches mail servers that appear operational but can't actually send or receive messages - the #1 silent failure mode that standard monitors miss.

‚ö°

Smart Rate Limiting

Login checks run every minute (cheap). Full roundtrip tests run hourly (expensive). Won't trigger anti-spam systems or abuse external providers.

üö¶

Stop-on-Failure

Halts at first error to protect your sender reputation. No bounces, no retries, no cascading failures. Clean failure indication.

üìä

Staleness Detection

Returns HTTP 418 when cached data is too old. Better to report DOWN than serve stale "UP" status from hours ago.

üê≥

Production Ready

Multi-platform images (amd64/arm64), Docker secrets support, non-root user, health checks, proper logging, graceful shutdown.

  mail-healthcheck:
    image: ghcr.io/0xa1bed0/mail-healthcheck:latest
    ports:
      - "3000:3000"
    environment:
      LOCAL_SMTP_HOST: smtp.your-server.com
      EXT_SMTP_HOST: smtp.gmail.com
    secrets:
      - mail_smtp_pass
    restart: unless-stopped
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mail-healthcheck

  replicas: 1
